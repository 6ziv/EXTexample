A Brief Introduction to EXTBack to menu

EXT,or Extended File System,was a file system designed for Linux.For more information,please follow this link.

In this simulate EXT system,the disk consists five parts:

First,4096 bits(512 bytes) that show whether the 4096 inodes have been taken.

Note that in a real EXT system it takes a integer instead of a bit to show how many links lead to an inode.

As I am not preparing to Links(especially hard links) in the system,I am using a bit showing whether the inode is occupied,to save space.

Second,4096 bits(512 bytes) that show whether the 4096 disk blocks have been taken.

(Yeah,it is a small disk..emmm)

Third,4096 blocks containing inode data,each 32 bytes.The inode data shows the information of files(and folders),and points out their data on the disk.

Fourth,4096 disk blocks,each 4096 bytes.For a folder,it contains its subitems,and for a file it contains its contents.

Last,16 bytes that server as a hash.It helps check whether the disk has been modified without authentication.

Also,if you are protection your disk with a password,this helps check whether the password is correct.

Only Chinese version available now.

Main parametersBack to menu

Filesize:

Each inode is associated with no more than five disk blocks,and each disk block takes 4096 bytes.

So the size of a file should not exceed 20480 bytes.

Security:

The disk is encrypted with AES-CTR.I'm trying to keep the influence on speed of encryption low,but when starting up,it takes a long while to scan through the disk,calculating hash of the disk.

For description,name the bytes 0,1,...

For the 16 bytes from 16*(n-1) to 16*n-1,we encrypt n with the master key to generate a subkey,and then outputs the subkey exclusive or the plain data.See also CTR mode.

Starting up:Back to menu

Compile:

First make sure that the requirements mentioned above are met.

Then check if make.exe (Or maybe nmake.exe if you use VS) exists.

If things goes all right,we can get started now.

We prepares two makefiles with the program.Choose one according to your OS.

Make it.(make -f makefile)

While make process running,it will ask you whether to activate the switches.

The first one is the self check switch.It serves as a debug switch (to test the compatibility of AES and MD5 algorithm whith your computer--as these two parts were not programmed by myself,I'm not sure if they suit other systems.)

The second one is the random IO switch.When it is activated,the program will do random IO tests when creating a new disk.(Only suggest activate it when there's a risk of bad tracks)

The third one is the password switch.When it is activated,the program will create disks with passwords.And you will need a password to open the disk.

Note that disk check will take place if and only if the password switch is on.

When the switch is off,the disk created can be open will have a null password.

Note that due to different control chars,a disk created in windows may fail to be open in Linux,and so may a disk created in Linux do in Windows.

Run:

On Windows:

./EXT.exe

On Linux:

./EXT

On First Run:

On first run,the program will generate a new file containing the disk.To avoid potential risks,it will fill this file with all-zero sequences,which may take a while.Personally I suggest a cup of coffee while waiting.

But please do not forget to input the password.And after inputting the password twice(for confirm),you will need to input it once more,to load the disk.

Logging Into The System:

When you have already loaded the disk,input command user root to log in.(When the disk is created,the program also creates user 'root' with no password.)

Usage:Back to menu

When the disk has been loaded,run command 'help' to show the list of commands.

Use help commandname(for example: 'help echo') to show the usage of a command.

(Well...due to laziness,I didn't add '--help' switch to commands.)

Note that commands and filenames and some other variables(almost everything) are case sensitive.

And that absolute paths must begin with '/'.

For introductions on output redirection,check 'help echo'.
